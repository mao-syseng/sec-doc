§ Terminology

OOC - Object Occurrence In CORE and DOMA it is each node in the tree structue. In SIMO it is the named boxes. Can also be called system.

OOR - Object Occurrence Relation AKA Interface The relationship between 2 OOC, shown in SIMO. Also called interfaces in SIMO. It is not the same as the relationship between 2 OOC in DOMA. This is called OOCR in some places, but that is wrong.

OODR - Object Occurence Domain Relation The connection between 2 OOC, shown in DOMA

Aspect | Context | Topnode This is what you select when using CORE, or select multiple when using DOMA

CORE - Common Reference Designation Editor This is a page in SEC-Hub, it shows a tree structure of Object Occurences where you can add new siblings or children to each Object Occurence

SIMO - Systems Integration Monitor This is a page in SEC-Hub, it shows an interactive N2 Diagram where users can define Object Occurence Relations which describe the input and output to/from other Object Occurences

DOMA - Domain Mapping This is a page in SEC-Hub and shows a tree structure of Object Occurences for each selected Aspect/Context/Topnode

Deadline | Milestone These should only be called deadline, but there are places in the code it is called milestone.
  • Are assigned to OORs, multiple different OORs can have the same deadline
  • Are created first without any link to any OORs.
